    Forget what you think you know... if you think the installation process is difficult. As long as you follow the detailed steps, you should be successful. If you're like me, you may feel apprehensive, when getting to know new software (having been burned in the past by poor documentation and sub-par applications).     My aim is quality and simplicity. Period. If we mean to accomplish great things, then first we must learn to jump the small hurdles. So let's start...

    1. Operating Systems

    Newfiles is designed to work on both Windows and Mac computers. Have one of these? You're good to go.

    A note to Linux users: You may still be able to look at these setup instructions and figure out how to get it running. But, I've never personally tried Newfiles on Linux.

    2. Required JDK Version

    Newfiles is programmed in Java and requires a minimum Java Development Kit (JDK) version. You can download the JDK for free from the internet (of course).

    The minimum requirement is JDK 1.7.0_17.

    If you already have this, or a later version, you can skip to step 3. Otherwise, you will have to download a JDK version that will meet this minimum requirement.

    3. Download Newfiles's Files

    There are two ways to download Newfiles's files...

    1. Via GIT repository (if you already know, love, and use GIT)

    If you choose to learn GIT, it will make it easier to get newer copies whenever Newfiles is updated.

    Knowing GIT is an essential skill to any type of project that requires code management. Code management is pretty important, man. I highly recommend learning how to use GIT, if you don't already. But I won't force you to use GIT. The choice is yours; I can't stop you from making poor decisions. But anyway, let's continue...
